Who I Am

About Me

I'm a Baltimore-based embedded engineer — someone who starts at the hardware and builds upward. My foundation is firmware: bare-metal C, peripheral drivers, real-time control loops, and the discipline that comes from debugging systems where a missed deadline or a wrong register write means a physical failure.

On top of that foundation I've built an edge ML practice. At Stanley Black & Decker I write firmware for next-generation DeWalt tools, implementing FOC motor control on ARM Cortex-M — and separately, I've built an end-to-end TinyML pipeline for an industrial drill: PyTorch activity detection model, compressed via structured pruning and INT8 quantization with ONNX, deployed using X-CUBE-AI. 95% precision in recoil detection in high-vibration environments. Before SBD, at BETiC, I built embedded prototypes in C for three medical devices: ventilation state machines for a clinical ventilator, PID temperature control for a scalp cooling system, and on-device stereo depth estimation for a dental intraoral scanner.

My M.Eng in Robotics at UMD is where the two sides fused: I led a team building a real-time UAV fault detection system — a hybrid SVM-autoencoder on 9+ sensor streams, Dockerized and running as live ROS nodes during actual flights, integrated directly into the autonomous navigation stack. That project is a good summary of how I work: embedded-first, then make it intelligent.
