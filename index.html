Who I Am

About Me

I'm an embedded engineer — someone who starts at the hardware and builds upward. My foundation is firmware: bare-metal C and C++ on ARM Cortex-M devices, I2C/SPI driver development, RTOS task scheduling, sensorless FOC motor control, and the kind of real-time debugging that lives on an oscilloscope and logic analyzer.

At Stanley Black & Decker I write firmware for DeWalt power tools — implementing sensorless and sensored FOC motor control, leading cross-tool systems integration, and programming DMA for current capture with bus-current protection. On the edge ML side, I've built an end-to-end TinyML pipeline for an industrial drill: a PyTorch activity detection model compressed via structured pruning and INT8 quantization with ONNX, deployed directly to a Cortex-M4 for on-device inference with strong precision in high-vibration environments.

Before SBD, at BETiC, I designed FreeRTOS architectures for medical device prototypes — preemptive scheduling with isolated task priorities for a ventilator, and ADC/I2C sensor integration for a scalp cooling system — all to ISO 13485 development standards. My M.Eng in Robotics at UMD fused the two sides: a hybrid SVM-autoencoder fault detection system deployed as live ROS nodes on a UAV during actual flights. Embedded-first, then make it intelligent.
